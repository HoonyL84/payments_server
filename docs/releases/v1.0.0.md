# Harness Template v1.0.0

범용 개인 프로젝트에서 사용할 수 있는 하네스 엔지니어링 템플릿의 첫 정식 릴리스다.

## 주요 기능

- Windows, macOS, Linux 공통 Node CLI와 호환용 Bash/PowerShell 래퍼
- backlog, active, archive 기반 티켓 수명 주기
- Quick/Full 검증 분리와 콘텐츠 지문 기반 완료 게이트
- 승인형 cleanup, 비파괴 recover, L4.5 auto-fix
- 선택형 L5 자율 실행과 API 예산·재시도·체크포인트
- opt-in 멀티에이전트 orchestration과 격리 worktree 통합
- GitHub Actions 크로스플랫폼 CI와 보안 스캔
- ESLint 정적 분석과 테스트 커버리지 최소 기준

## 품질 기준

- `npm test`
- `npm run coverage`
- `npm run lint`
- `npm run build`
- `npm run harness -- verify --full`

초기 커버리지 게이트는 현재 전체 CLI 테스트 기준인 lines/statements 65%, functions/branches
70%로 시작한다. 신규 모듈은 독립 단위 테스트를 함께 추가하고, 전체 기준은 회귀 없이
점진적으로 상향한다.

## 알려진 제한

- L5와 멀티에이전트는 명시적으로 활성화해야 하는 Experimental 기능이다.
- 실제 provider API 품질과 장시간 안정성은 provider, 모델, 호스트 권한에 영향을 받는다.
- Git 기반 상태 추적을 사용하므로 Git 저장소와 Node.js 20 이상이 필요하다.
